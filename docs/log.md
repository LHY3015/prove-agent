# Phase retrospectives (5 lines per phase).

## Phase 1 — Baseline pipeline (A0)
- Built the full no-skills loop: datagen (8 formats/4 layouts, WeasyPrint→pdfplumber), router fingerprint, LLM extraction, deterministic validator, fingerprint-keyed sample pool, SQLite traces, LangGraph pipeline, A0 ablation runner. 26 tests green, ruff clean.
- Key design call: the "bucketized bbox" fingerprint was unstable within a format (variable item rows / content-driven column widths); replaced with a header-region token+position fingerprint that cuts above the items table — stable per format, Jaccard 0.4 across formats, and it aligns with the P4 attribution rules (below-header drift keeps routing → data_drift, not routing_error).
- Added `llm_client.py` (Fake/OpenAICompat) and `layout.py` (shared PDF→text_layout) — neither was in the plan's tree but both are load-bearing; fake transport is what keeps CI key-free.
- Fable review at the phase gate: 6/7 decisions endorsed; applied its 3 fixes — pinned Python 3.12, froze the trace routing contract (`route_confidence`=similarity to executed format + raw fingerprint hash on every trace), versioned `text_layout` as the skill ABI.
- Deferred honestly: no-`--live` A0 numbers are simulated (labeled), real curves await a Qwen `--live` run; value normalization deferred (ground truth = displayed strings) so the Phase-2 challenge is positional localization.

## Phase 2 — Skill synthesis + admission (the core loop)
- Built the full learn-to-compile loop: sandbox (isolated `python -I` subprocess, rlimits, import whitelist, no-net-path, wall-timeout — security-tested), synthesis agent (LLM writes `extract()`, self-repair loop in the sandbox), registry (SQLite state machine + discounted-Beta ledger), admission gate (persistent held-out split, sandbox F1 vs the pool's own validated fields), pipeline skill-routing + imperative `_lifecycle` orchestration, ablations A1/A2. 52 tests green, ruff clean.
- Headline result (simulated, key-free): cost-per-doc 242→97 tok as skills come online; A1 (no gate) admits a memorization-overfit skill that emits **15 silent, validation-passing wrong-field docs**, while A2's gate rejects the identical candidate and resynthesizes a good one — the A1-vs-A2 silent-failure demo the project exists to show.
- CI stays key-free via `evals/fake_skills.py`: one hand-written generic parser hits 100% field accuracy on all 8 formats (their field *values* share formats even though labels differ), and a `FakeSynthesizer` emits the identical synthesis stream to both A1 and A2 so only the gate differs (Fable's block-level fix).
- Fable-cleared twice: pre-build architecture ruling (sandbox `python -I`, imperative lifecycle, memo-keyed-on-full_text overfit, persistent holdout, parsed money/date comparison) and a phase-gate GO-WITH-FOLLOWUPS. Mid-build change: dropped strict-zero-mismatch@holdout≤3 for uniform `F1≥0.95 ∧ no-crash` — strict × persistent-holdout × noisy self-supervised oracle = permanent format rejection (hit at 4% error); Fable signed off (admission now catches *systematic* wrongness; monitor/audit are the backstop).
- Carried to Phase 3 (design conditions from the gate): campaign-scoped rejection reset + frozen-holdout/pool invalidation on deprecation-triggered resynthesis (else drift self-healing is structurally broken); calibrate routing-noise against the 20-doc/0.2 monitor window so A2's wrongful-kill actually fires; monitor must cover trial skills.

## Phase 3 — Continuous monitoring + self-healing
- Closed the self-healing loop end-to-end: `monitor.py` (per-skill_id sliding window + discounted-Beta floor), deprecation → LLM fallback → pool re-accumulation → resynthesis → re-admission, driven by a mid-stream date-format drift. `scenarios/drift_demo.py` produces the timeline plot with zero manual steps; 60 tests green, ruff clean. Headline (key-free): v1 admitted → drift at doc 30 → deprecated at doc 35 via `failure_batch:window(5/20)` → v2 admitted at doc 45 → healed.
- Fable pre-build design review (GO-WITH-CHANGES) shaped six mechanics: pool `invalidate` is a **tombstone** (rows kept for P4 attribution/audit, excluded from live count/synthesis), monitor windows keyed by **skill_id** (a resynthesized skill starts clean), trial→active promotion guarded against `deprecated` in both pipeline and registry (kill-vs-promote race), a fast-path **absolute failure floor** (`min_failures`=3, `min_window`=10) so routing noise can't kill a young skill, and a loud `campaign_exhausted` event instead of silent stranding.
- Self-heal invalidation is the crux: on deprecation the pipeline tombstones the stale pool, drops the frozen admission holdout, and opens a new synthesis campaign (rejections reset, campaign id logged). Resynthesis then re-fires only once the pool re-reaches the trigger with FRESH post-drift samples — that trigger-over-a-cleared-pool *is* the freshness gate. Without it, resynthesis would fire immediately on stale pre-drift training and re-deprecate in a loop.
- Drift is fingerprint-stable by design: the router drops digit/month tokens, so a date-format change keeps routing to the existing skill (genuine template drift, not new-format discovery) → the skill's baked-in date handling misses → `missing_field:invoice_date` → validation fails deterministically. Key-free honesty: the `SpecializedSynthesizer` double learns its date competence only from its training samples (a real LLM shown one style writes a parser for that style), so the pre-drift skill genuinely fails on the new style and the resynthesized one heals; `--live` carries the real science.
- A2-noise calibration (a P3 DoD item) is asserted, not just chosen: `test_monitor` proves the fast path fires on a failure batch but steady ~10% noise never trips (floor + threshold), and the drift integration test asserts the deprecation came from the window path, not the ledger floor. Documented restart caveat: `_synth_state`/`_holdout_ids`/monitor windows are in-memory while the registry is SQLite — a mid-campaign restart re-chooses holdouts and forgets rejection counts (acceptable for the demo; persistence deferred).
